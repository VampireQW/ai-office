# 产研工作流体系（PE Workflows）

> 升级自 `pmworkflows/`，覆盖独立软件项目从 0 到 1 的完整研发链路。
> 创建于 2026-02-14

## 流程总览

```
市场调研 → 竞品分析 → PRD文档 → 架构设计 → SDD任务拆解 → 代码实现 → 测试用例
```

## 命令列表

| 命令 | 说明 |
|------|------|
| `/pe-full` | 一键执行全流程（每阶段暂停确认） |
| `/pe-research` | 市场调研：市场规模、用户画像、技术可行性预研 |
| `/pe-competitive` | 竞品分析：功能对比、技术栈对比、差异化策略 |
| `/pe-prd` | PRD文档：需求详述、产品框架图、验收标准、演示数据规格 |
| `/pe-architecture` | 架构设计：信息架构、数据模型、API设计、技术选型 |
| `/pe-tasks` | SDD任务拆解：规格编号、任务拆解、追踪矩阵 |
| `/pe-code` | 代码实现：**可交付1.0版本**（非DEMO） |
| `/pe-testing` | 测试用例：功能/交互/边界测试 |
| `/pe-change` | SDD变更控制：变更请求、影响分析、规格同步 |
| `/pe-summary` | 项目总结：交付状态、覆盖率、风险和结论 |
| `/pe-overview` | 查看工作流总览 |

## 与旧版 PM 工作流的区别

| 维度 | PM 工作流 `/pm-xxx` | PE 工作流 `/pe-xxx` |
|------|-------------------|-------------------|
| 定位 | 产品经理视角 | 产研一体（产品+工程） |
| 阶段 | 6 阶段（含原型设计） | 7 阶段（去原型，加 SDD 任务和测试） |
| 代码产出 | 演示级 DEMO | 可交付 1.0 版本 |
| 演示数据 | 简单模拟 | ≥20 条/场景，真实数据 |
| 分支交互 | 仅核心路径 | 正常/异常/边界全覆盖 |
| 状态管理 | 未强制 | loading/success/empty/error 四态 |
| 规格追踪 | 未强制 | REQ/FR/AC/TASK/TC 端到端追踪 |

## SDD 核心标准

1. **规格唯一源** — `3-PRD.md` 必须包含稳定编号：REQ/FR/NFR/AC/DATA/RISK
2. **规格先冻结** — 进入代码前 PRD 状态必须为 Frozen，变更进入变更记录
3. **任务可追踪** — `5-开发任务.md` 中每个 TASK 必须关联 FR/AC/NFR/DATA
4. **测试可回溯** — 每个测试用例 TC 必须对应 AC 或 NFR
5. **矩阵闭环** — 追踪矩阵覆盖：需求 → 验收标准 → 任务 → 实现文件 → 测试用例
6. **变更受控** — Frozen 后的新需求必须走 CHANGE，不得直接进入代码
7. **总结可核验** — `PROJECT_SUMMARY.md` 的数字必须来自规格、任务、测试和变更记录

## 代码实现核心标准

1. **数据丰富** — 列表 ≥20 条，中文真名，多状态混合，禁止假数据
2. **交互完整** — CRUD 闭环、搜索筛选、二次确认、表单校验
3. **四态完备** — 每个数据区域：loading / success / empty / error
4. **视觉 Premium** — 渐变、微动画、响应式、设计系统变量
5. **零报错** — 无 JS 控制台错误
